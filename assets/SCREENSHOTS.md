# Screenshots Guide

This directory contains all screenshots and visual assets used in the README.md file.

## Screenshot Requirements

### File Naming Convention
- Use lowercase with hyphens: `main-editor.png`
- Be descriptive: `ai-assistant-panel.png` instead of `screenshot1.png`

### Image Specifications
- **Format**: PNG or JPG
- **Resolution**: At least 1920x1080 for desktop screenshots
- **File Size**: Optimize images to be under 500KB each
- **Aspect Ratio**: 16:9 preferred for consistency

## Required Screenshots

Add the following screenshots to the `screenshots/` folder:

1. **main-editor.png** - Main editing interface with AI suggestions panel
2. **ai-assistant.png** - AI-powered writing suggestions and improvements
3. **dashboard.png** - Document management dashboard
4. **collaboration.png** - Real-time collaboration features
5. **themes.png** - Light and dark theme options
6. **export.png** - Export options dialog

## Optional Assets

- **logo.png** - Application logo (recommended size: 512x512px)
- **demo.gif** - Animated demo of key features
- **architecture.png** - System architecture diagram

## Image Optimization

Before adding screenshots, optimize them using tools like:
- [TinyPNG](https://tinypng.com/)
- [ImageOptim](https://imageoptim.com/)
- [Squoosh](https://squoosh.app/)

## Tips for Great Screenshots

1. **Clean UI**: Remove any personal information or test data
2. **Consistent Theme**: Use the same theme across all screenshots
3. **Highlight Features**: Show the most important features clearly
4. **High Quality**: Ensure text is readable and images are sharp
5. **Context**: Include enough context to understand what's being shown
6. **Annotations**: Consider adding arrows or highlights to draw attention to key features

## Adding Screenshots to README

The README.md file already includes placeholders for these screenshots. Simply add your images to the `screenshots/` folder with the correct filenames, and they will automatically appear in the README.
